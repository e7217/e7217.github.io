# Blog UI/UX Refresh Design

## Goal

개발 기록용 기술 블로그의 화면을 읽기 중심으로 정리한다. 방문자가 첫 화면에서 블로그의 주제를 빠르게 파악하고, 글 목록에서 읽을 글을 쉽게 고르며, 본문에서는 긴 기술 기록을 부담 없이 따라갈 수 있어야 한다.

## Current Issues

- 홈 화면의 소개 문단과 주제 목록이 기본 문서처럼 보여서 첫 화면의 정보 위계가 약하다.
- 글 목록은 제목, 메타, 요약의 구분이 흐려 빠르게 훑기 어렵다.
- 본문 제목, 메타 정보, 태그, 이전/다음 버튼의 크기와 간격이 서로 다른 기준처럼 보인다.
- 사이드바와 네비게이션은 기능은 충분하지만 본문보다 시각적으로 도드라지는 지점이 있다.

## Approaches Considered

### 1. Theme Preserving Refresh

Minimal Mistakes 구조를 유지하고 `index.html`의 홈 콘텐츠와 `assets/css/main.scss`만 조정한다.

- 장점: GitHub Pages/Jekyll 빌드 안정성이 높고 변경 범위가 작다.
- 단점: 테마가 제공하는 HTML 구조의 제약은 남는다.
- 판단: 이번 단계의 권장안이다.

### 2. Custom Layout

홈, 글 목록, 본문 레이아웃을 커스텀 Liquid 템플릿으로 분리한다.

- 장점: UI 제어권이 커진다.
- 단점: 유지보수 범위가 늘고, 초기 블로그 단계에서는 과하다.
- 판단: 글이 더 많이 쌓인 뒤 검토한다.

### 3. Full Design System

디자인 토큰, 컴포넌트, 별도 레이아웃 체계를 만든다.

- 장점: 장기적으로 일관성이 가장 좋다.
- 단점: 현재 콘텐츠 규모에 비해 비용이 크다.
- 판단: 제외한다.

## Selected Design

이번 변경은 `Theme Preserving Refresh`로 진행한다.

홈 화면은 마케팅 페이지가 아니라 개발 기록의 인덱스로 다룬다. 히어로는 작고 차분하게 유지하고, 본문 상단에 블로그의 기록 원칙과 큰 주제만 보여준다. 주제는 카드형 반복 요소로 정리하되, 장식보다 스캔성을 우선한다.

글 목록은 각 글을 독립적인 항목으로 보이게 만든다. 제목, 읽는 시간, 요약이 한눈에 구분되도록 여백과 경계를 부여하고, hover 상태는 링크 발견성을 높이는 수준으로만 둔다.

본문은 14px 본문 기준을 유지한다. 제목 계층, 코드 블록, 인용문, 표, 태그, 공유 버튼, 페이지네이션의 크기와 간격을 본문 스케일에 맞춰 정리한다.

- 홈, 글 목록, 대표 글 화면을 데스크톱 캡처로 확인한다.
- GitHub Pages 빌드가 `built` 상태가 되는지 확인한다.
